# YouTube 下载器：深度使用说明与功能手册

本程序是基于 `yt-dlp` 与 `ffmpeg` 开发的高性能桌面下载工具，支持单视频下载、批量下载（播放列表/频道）、下载队列管理、历史记录查询等功能。本手册旨在帮助您全面掌握从基础参数到高级后期处理的所有功能。

---

## 1. 核心操作流程

### 📺 单视频下载流程

1. **输入链接**：将 YouTube 视频 URL 粘贴至输入框，支持以下链接格式：
   - 单视频：`https://www.youtube.com/watch?v=xxxxx`
   - 短链接：`https://youtu.be/xxxxx`
   - 嵌入链接：`https://www.youtube.com/embed/xxxxx`

2. **获取视频信息**：
   - **🔍 获取分辨率/格式**：点击后程序会在线解析该视频支持的所有画质组合，解析完成后可进行手动筛选格式。
   - **⚡ 直接下载**：根据当前选择的"下载策略"自动匹配最佳画质并直接下载，无需手动选择格式。

3. **添加到队列**：点击"添加到队列"按钮，任务将进入下载队列。

4. **启动下载**：切换到"下载队列"标签页，点击"▶ 开始全部"启动下载任务。也可单独选中某个任务点击"▶ 开始"按钮启动。

### 📋 批量下载流程

1. **选择来源类型**：在批量下载页面，可选择以下来源类型：
   - **播放列表 (Playlist)**：YouTube 播放列表 URL
   - **频道 (Channel)**：YouTube 频道 URL
   - **上传列表 (Uploads)**：频道的上传视频列表

2. **输入链接并获取条目**：粘贴播放列表或频道 URL，点击"获取条目"按钮。程序会解析并列出所有可下载的视频条目。

3. **筛选与选择**：
   - 在条目列表中勾选需要下载的视频
   - 支持"全选"、"清空全部"、"保留可用"等快捷操作
   - 可按标题搜索过滤特定视频

4. **设置批量下载参数**：
   - **并发数**：同时下载的任务数，建议 2-4 个
   - **下载策略**：选择统一的画质策略应用于所有选中条目
   - **子文件夹**：可设置按播放列表名称或频道名称自动创建子文件夹

5. **添加到队列**：点击"添加选中任务"将选中的视频添加到下载队列。

## 2. 基础设置与性能调节

### 📁 存储与重试设置

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **保存路径** | 用户选择 | `~/Downloads` | 下载文件的保存目录，点击"📂 浏览/设置"可更改，建议选择空间充足的磁盘 |
| **重试次数 (Retries)** | 0 - 10 | `3` | 针对网络不稳定导致的下载中断，程序自动重新尝试的次数。设置为 0 则不重试 |

### 🚀 下载调优参数

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **并发数** | 1 - 10 | `2` | 同时下载的任务数。建议设为 `2-4`，过高可能导致 YouTube 暂时封锁 IP |
| **限速 (MB/s)** | 0 - 100 | `2` | 限制**单个任务**的最大下载速度。设置 `0` 为不限速。下载时若需同时看网页视频，建议开启限速以保障带宽 |

### 📥 下载策略 (预设格式)

程序提供多种预设下载策略，应用于"直接下载"或批量下载场景：

| 预设名称 | 说明 | 适用场景 |
|----------|------|----------|
| **最佳画质 (best)** | 选择最高画质和音质的组合 | 追求最高画质 |
| **最佳兼容 (best compat)** | 选择画质与兼容性最佳的组合，确保能在各种播放器播放 | 需要广泛兼容性 |
| **仅音频 (audio only)** | 仅下载音频流，保存为 MP3 或 M4A | 音乐下载、有声内容 |
| **1080p** | 限制最高分辨率为 1080p | 节省磁盘空间、网络带宽有限 |
| **4K** | 优先选择 4K 分辨率 | 高清大屏观看 |
| **H.264 兼容** | 强制使用 H.264 视频编码 | 需要在老旧设备播放 |
| **手动选择** | 不使用预设，在格式列表中手动选择具体格式 | 特殊需求 |

## 3. 格式筛选与手动模式

在点击"获取分辨率/格式"后，程序会解析并显示该视频所有可用的格式组合。此时可使用以下筛选和排序功能：

### 🔍 格式筛选选项

| 筛选选项 | 说明 |
|----------|------|
| **仅 MP4** | 仅显示 .mp4 容器格式的视频流 |
| **仅带音频** | 仅显示内置音轨的视频流（无需额外合并音视频） |
| **仅 60fps** | 仅显示 60 帧及以上的高帧率视频 |
| **仅 4K+** | 仅显示 4K 及以上超高清分辨率 |
| **仅音频轨** | 仅显示不含画面的独立音轨（用于提取音频） |

### 📊 排序方式

| 排序方式 | 说明 |
|----------|------|
| **按画质 (Quality)** | 按视频分辨率和帧率降序排列 |
| **按大小 (Size)** | 按预估文件大小降序排列 |

### 🎯 手动格式选择

选中"手动选择格式"策略后，可在"手动格式"下拉框中指定具体的 Format ID（由 yt-dlp 定义，如 `137+140` 表示视频 137 与音频 140 合并）。此功能适用于需要精确控制输出格式的高级用户。

## 4. 后处理增强功能 (Post-Processing)

程序提供丰富的后期处理选项，勾选后在下载完成后自动执行：

### 📷 媒体元数据处理

| 选项 | 说明 |
|------|------|
| **☑ 嵌入封面** | 将视频封面图直接写入视频文件（作为文件缩略图），支持 MP4、MKV 等容器 |
| **☑ 嵌入元数据** | 写入标题、作者、发布日期、描述信息等到视频文件 |
| **☐ 写入缩略图文件** | 在视频同级目录下保存一张 `.jpg` 封面图片文件 |
| **☐ 写入信息 JSON** | 保存视频所有技术参数的 JSON 元数据文件 |
| **☐ 写入描述** | 将网页上的视频详细说明保存为 `.txt` 文件 |
| **☐ 章节写入** | 将视频内部的时间轴进度章节信息写入 `.chapters.txt` 文件 |

### 🔧 视频处理选项

| 选项 | 说明 |
|------|------|
| **☐ 保留中间视频** | 合并音视频后，不删除原始的临时流文件（用于调试） |
| **☐ H.264 兼容模式** | 强制将视频转码或封装为 H.264 编码，确保在所有播放设备上均可播放（会显著增加下载时间） |

### 📁 文件命名与归档

| 选项 | 说明 |
|------|------|
| **自定义文件名** | 支持使用变量自定义输出文件名，如 `%(title)s_%(id)s.%(ext)s` |
| **使用视频 ID 命名** | 在文件名中包含视频 ID 以避免重名 |
| **归档记录** | 启用后自动记录已下载视频 ID，避免重复下载 |

## 5. 认证与反爬机制

### 🍪 Cookie 认证

用于解决"年龄限制"、"会员内容"或"403 访问受限"等问题。

**使用方法**：
1. 在浏览器中安装 Cookie 导出插件（如 "Get cookies.txt LOCALLY"）
2. 登录 YouTube 账号后，导出 `www.youtube.com` 域名的 Cookie 为 `.txt` 格式
3. 将导出的 Cookie 文件重命名为 `www.youtube.com_cookies.txt` 并放入程序同目录
4. 程序会自动检测并使用该 Cookie 文件

**Cookie 文件格式**：必须是 Netscape 格式的 Cookie 文件，每行包含域名、是否 HttpOnly、是否安全、过期时间、名称、值等字段。

### 🛡️ PO Token 机制

应对 YouTube 系统针对机器人请求的严格检测，启用后可显著降低大规模解析时的 403 报错风险。

**前置要求**：
- 系统需安装 Node.js（建议 v16 及以上）
- 程序目录下的 `tools/po_token/` 文件夹需包含 `generate_token.js` 脚本

**工作原理**：
- PO Token 是 YouTube 用于验证请求来源的加密令牌
- 程序会自动调用 Node.js 脚本生成有效 Token
- Token 会被缓存一段时间后自动刷新

**状态查看**：
- 程序顶部状态栏显示 PO Token 当前状态
- 可通过"运行状态"窗口查看详细状态信息

## 6. 下载队列管理

### 队列操作

| 操作 | 说明 |
|------|------|
| **▶ 开始全部** | 启动队列中所有待下载的任务 |
| **⏹ 停止全部** | 停止所有正在下载的任务 |
| **🗑 清空已完成** | 清除已完成（成功/失败）的任务记录 |
| **🔄 重试失败** | 重新尝试下载失败的任务 |

### 单任务操作

在队列列表中选中任务后，可执行以下操作：

| 操作 | 说明 |
|------|------|
| **▶ 开始** | 启动选中的单个任务 |
| **⏹ 停止** | 停止正在下载的任务 |
| **🔄 重试** | 重新下载该任务 |
| **🗑 删除** | 从队列中删除该任务 |

### 任务状态

| 状态 | 说明 |
|------|------|
| **等待中** | 任务已添加，等待被调度执行 |
| **下载中** | 正在下载该视频 |
| **已完成** | 下载成功完成 |
| **已失败** | 下载失败（显示失败原因） |
| **已停止** | 用户手动停止 |

## 7. 历史记录管理

### 查看历史

程序会自动记录下载历史，支持以下查看方式：

- **下载成功记录**：记录所有成功下载的视频信息
- **下载失败记录**：记录所有失败的任务及失败原因

### 历史记录功能

| 功能 | 说明 |
|------|------|
| **查看详情** | 双击历史记录可查看详细信息（URL、下载时间、文件大小等） |
| **重新下载** | 可从历史记录中快速重新发起下载 |
| **清空历史** | 支持清空全部历史或按条件筛选清除 |
| **导出记录** | 历史记录支持导出为 JSON 格式 |

### 存储方式

历史记录采用双存储机制：
- **SQLite 数据库** (`download_history_ytdlp.sqlite3`)：用于高效查询和大量历史记录存储
- **JSON 文件** (`download_history_ytdlp.json`)：用于备份和迁移

## 9. 环境依赖与故障排查

### 必需依赖

| 组件 | 说明 | 放置位置 |
|------|------|----------|
| **yt-dlp.exe** | 负责解析与下载的核心引擎 | 程序同目录 |
| **ffmpeg.exe** | 负责合并视频、提取音频、写封面及转码 | 程序同目录或系统 PATH |

### 常见问题与解决方案

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| **没有声音** | 缺少 ffmpeg，导致无法将独立下载的音视频轨合并 | 确保 ffmpeg.exe 存在于程序目录 |
| **解析失败** | 网络问题、URL 错误、或 YouTube 反爬机制 | 尝试更新 yt-dlp，或使用 Cookie 辅助验证 |
| **403 错误** | YouTube 反爬检测 | 启用 PO Token，或使用 Cookie 登录 |
| **年龄限制** | 视频需要登录或会员权限 | 导入有效的 YouTube Cookie |
| **下载速度慢** | 网络带宽受限或 YouTube 限速 | 尝试使用代理，或降低并发数 |
| **格式获取失败** | 视频在特定地区不可用 | 检查网络连接，尝试使用代理 |

### 更新程序

- **更新 yt-dlp**：点击程序右上角的"更新 yt-dlp"按钮，程序会自动下载最新版本的 yt-dlp.exe
- **更新日志**：程序会在启动时检查更新，并在下载目录生成更新日志

## 10. 高级使用技巧

### 使用代理

在系统网络设置代理后，yt-dlp 会自动使用系统代理。如需单独配置，可在环境变量中设置 `HTTP_PROXY` 和 `HTTPS_PROXY`。

### 批量下载节流

对于大型播放列表或频道批量下载，建议：
1. 将并发数设置为 1-2
2. 启用限速（如 1-2 MB/s）
3. 适当增加重试次数
4. 避免短时间内大量请求

### 文件名模板变量

自定义文件名时支持以下变量：

| 变量 | 说明 |
|------|------|
| `%(title)s` | 视频标题 |
| `%(id)s` | 视频 ID |
| `%(uploader)s` | 上传者名称 |
| `%(upload_date)s` | 上传日期 (YYYYMMDD) |
| `%(ext)s` | 文件扩展名 |
| `%(resolution)s` | 分辨率 |
| `%(duration)s` | 时长（秒） |
| `%(playlist_index)s` | 播放列表中的序号 |
