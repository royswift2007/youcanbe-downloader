# YouTube 下载器：深度使用说明与功能手册

本程序基于 `yt-dlp` + `ffmpeg` +（可选）`deno`，支持单视频下载、批量下载（播放列表/频道）、下载队列管理、历史记录查看、媒体工具处理、组件中心、认证状态中心、运行状态中心、Hook 扩展等功能。本手册以当前代码实现为准，逐项给出参数默认值、取值范围与使用方式。

> [!IMPORTANT]
> 首次运行前/首次运行时，请先进入 **设置 → 组件中心**，点击“**组件更新（yt-dlp/ffmpeg/deno）**”。
> 程序依赖这些组件完成下载与处理；如果本地还没有组件，请先更新再使用。
> YCB 运行时只识别程序目录内的 `yt-dlp / ffmpeg / deno`，不会使用系统 `PATH` 中同名程序。
> 安装器中的组件检测、程序内的组件中心，以及运行状态中心的判断口径也都相同：只按程序目录内组件判断。

## 0. 安装与运行权限说明

> [!CAUTION]
> 如果您将程序按照默认设置安装在 **`C:\Program Files`** 或 **`C:\Program Files (x86)`** 等系统目录下：
> 1. **权限需求**：程序在运行时需要产生日志（`logs/`）、下载历史（`download_history_ytdlp.json`）及配置文件（`window_pos.json`）。这些文件的正常读写是程序稳定运行的前提。
> 2. **自动授权**：当您使用官方提供的 `.exe` 安装包安装时，安装程序会自动为该目录授予运行所需的读写权限。
> 3. **手动移动**：如果您从压缩包手动解压、或后期手动移动了安装文件夹，请确保当前用户对该文件夹具有 **完全控制权限**，否则可能会因“Access is denied（拒绝访问）”导致无法下载或报错。

## 1. 核心操作流程

### 1.0 程序功能总览

当前版本已经覆盖以下几大功能区域：

| 序号 | 功能说明 |
|------|----------|
| 1 | **单视频下载**：支持 YouTube / Generic 两种模式，支持自动策略、手动选格式、直接下载、加入队列、字幕、后处理、代理、Cookies、区段下载等。 |
| 2 | **批量下载**：支持播放列表 / 频道条目解析、条目筛选、批量统一参数、批量手动策略、批量入队。 |
| 3 | **下载队列**：支持任务排队、开始 / 重试、停止选中、停止全部、删除选中、清除完成、查看实时日志。 |
| 4 | **历史记录**：支持查看已完成/失败历史摘要，并可手动刷新与清空。 |
| 5 | **认证状态中心**：查看 Cookies 文件、Browser Cookies、PO Token 状态、最近认证诊断与建议动作。 |
| 6 | **运行状态中心**：查看当前任务概览、历史数据库状态、最近运行问题摘要与详情。 |
| 7 | **组件中心**：查看 `yt-dlp / ffmpeg / deno` 状态、路径、版本，并执行一键更新和导出诊断。 |
| 8 | **媒体工具**：执行本地媒体处理任务，例如抽音频、裁剪、拼接、烧录字幕、缩放、裁切、旋转、水印、响度归一化等。 |
| 9 | **全局设置**：管理语言、剪贴板监听、自动解析、Cookies 模式、保存目录、并发、限速、重试等全局参数。 |
| 10 | **扩展能力**：支持 Hook、运行期诊断、日志高亮、高级参数白名单控制、失败摘要记录等。 |

> 建议第一次使用时，先按“组件准备 → 设置检查 → 单任务验证 → 批量使用 → 媒体工具”的顺序逐步熟悉。

### 📺 单视频下载流程（YouTube / Generic）

1. **选择模式**：
   - **YouTube 模式**：仅支持 YouTube 链接，支持获取格式列表与手动选择格式。
   - **Generic 模式**：支持任意合法 http/https 链接（含域名），不提供 YouTube 格式列表。

2. **输入链接**：
   - YouTube 支持格式示例：
     - `https://www.youtube.com/watch?v=xxxxx`
     - `https://youtu.be/xxxxx`
     - `https://www.youtube.com/embed/xxxxx`
   - Generic 模式支持任何 http/https URL（必须包含域名）。

3. **获取视频信息**：
   - **🔍 获取分辨率/格式**：仅 YouTube 模式可用。解析该视频支持的所有格式组合，并显示表格供筛选/手动选择。
   - **⚡ 直接下载**：无需手动选格式，按“下载策略”自动匹配并下载。

4. **添加到队列**：
   - 点击“添加到队列”后进入下载队列，等待手动开始或批量开始。

5. **开始下载**：
   - 在“下载队列”页点击“开始全部”，或选择单个任务使用“开始 / 重试”。

**本次更新补充**
- “仅音频 (audio only)”预设现在可以直接通过“直接下载”启动，不再要求先手动选择音频格式。
- 单视频“仅音频”路径会按音频抽取方式执行，避免把音频下载任务误走视频合并流程。

---

### 📋 批量下载流程（播放列表 / 频道）

1. **输入链接并获取条目**：
   - 粘贴播放列表/频道 URL，点击“获取条目”。

2. **筛选与选择**：
   - 支持“全选 / 清空全部 / 保留可用”。
   - 支持“隐藏不可用 / 仅 Shorts”。

3. **设置批量下载参数**：
   - 统一应用下载策略、输出格式、字幕/后处理、并发/限速、节流与代理等参数。

4. **添加选中任务**：
   - 点击“添加选中任务”，加入下载队列。

**本次更新补充**
- 批量页视频输出格式现支持 `mp4 / mkv / webm`。
- 批量页新增“启用手动”与“手动设置”，可基于样本视频的实际格式，为整批任务生成规则型 `-f` 表达式。
- 批量手动策略支持预设1、预设2，以及可选的“最终兜底链”。

## 2. 基础设置与性能调节（全局设置页）

### ⚙️ 设置页（Settings）

|------|--------|------|----------|

| 自动解析 | 关闭 | 剪贴板监听触发后自动执行“获取分辨率/格式”（仅 YouTube 模式有效） | 开启后复制 YouTube URL 即可自动解析 |

#### 2.3 语言切换

|------|--------|----------|------|----------|

#### 2.4 快捷入口

| 入口 | 功能 | 说明 |
|------|------|------|
| 认证状态 | 打开认证状态中心 | 查看 Cookies/PO Token 状态与建议处理 |
| 历史记录 | 打开历史记录窗口 | 展示历史列表（文本形式） |
| 组件中心 | 打开组件中心 | 查看 yt-dlp/ffmpeg/deno 版本与状态 |
| 运行状态 | 打开运行状态中心 | 查看任务队列、DB 状态、最近运行问题 |

**补充说明**
- 这些入口本质上都是“状态/说明类辅助窗口”，适合在主流程之外做检查、排障和复核。
- 当你遇到“能打开程序但不知道哪里有问题”时，优先打开的顺序通常建议为：`组件中心 -> 认证状态中心 -> 运行状态中心 -> 下载队列日志`。
- “历史记录”和“使用介绍”属于偏查看型窗口；“认证状态 / 运行状态 / 组件中心”更偏诊断型窗口。
- 顶部区域和设置页中的这些入口本质上指向同一类功能，只是进入位置不同。

> 首次运行时，程序目录可能没有 `yt-dlp.exe / ffmpeg.exe / deno.exe`。
> 请进入 **设置 → 组件中心**，点击“**组件更新（yt-dlp/ffmpeg/deno）**”按钮，程序会自动下载并安装到根目录，下载过程会显示进度。

#### 2.5 存储与性能

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 保存路径 | `~/Downloads` | 用户选择 | 下载文件默认保存目录 |
| 重试次数 | `3` | `0 - 10` | 网络不稳定时自动重试；`0` 表示不重试 |
| 并发数 | `2` | `1 - 10` | 建议 `2 - 4`，过高可能增加站点风控概率 |
| 限速 (MB/s) | `2` | `0 - 100` | 单任务限速；`0` 表示不限速 |

**补充说明**
- 设置页的并发、重试、限速会作为各下载页初始化值来源；单视频页和批量页仍可在各自页面中单独调整。
- 保存路径会影响下载任务默认输出目录，也会影响历史记录中的路径显示以及“打开目录”按钮的行为。
- 如果正在运行任务，修改这些参数通常只影响后续新建任务；已在队列中的任务是否立即采用新值，以其入队时保存的任务配置为准。
- 对 YouTube 站点而言，较高并发和较激进的节流参数更容易触发风控，推荐优先保证成功率而不是盲目提高速度。

## 3. 单视频页参数（YouTube / Generic）

### 3.1 模式与 URL

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 模式 | `youtube` | `youtube` / `generic` | YouTube 模式可获取格式；Generic 模式为通用下载 |
| 视频 URL | 空 | 合法 URL | YouTube 模式仅支持 YouTube/youtu.be 链接；Generic 支持任意 http/https |

**按钮与动作**：
- **获取分辨率/格式**：仅 YouTube 模式可用，解析格式表。
- **直接下载**：按策略直接启动下载任务。
- **添加到队列**：加入下载队列，稍后统一启动。

**补充说明**
- 当前单视频页 URL 输入框为 2 行高度，便于粘贴较长链接、带参数链接或临时多行整理内容。
- 在 YouTube 模式下，如果链接不是标准 YouTube 域名，界面会直接阻止获取格式和入队。
- 在 Generic 模式下，“获取分辨率/格式”会被禁用，因为通用站点通常无法稳定提供统一格式表。
- 剪贴板监听与自动解析开启后，复制 YouTube 链接可自动填入此区域，减少手动粘贴步骤。

### 3.2 视频信息卡

| 字段 | 说明 |
|------|------|
| 标题 | 解析后显示视频标题 |
| 元信息 | ID / 频道 / 时长 / 观看 / 上传 / 语言 |

**补充说明**
- 在 YouTube 模式点击“获取分辨率/格式”后，通常会先刷新这里的标题和元信息，再更新下方格式表。
- 如果当前仍显示“尚未解析视频”或默认占位元信息，说明该链接尚未成功完成元数据读取。
- 当已经成功获取标题、且你没有手动填写“重命名”时，加入队列后的任务名称会优先使用这里解析到的视频标题，而不是简单的预设名称。
- Generic 模式下，这里更多承担“标题占位与状态反馈”的作用，信息完整度取决于站点是否支持元数据解析。

### 3.3 下载策略（预设）

| 预设 | 说明 | 实际格式（yt-dlp） |
|------|------|------------------|
| 最佳画质 | 最高画质与音质组合 | `bestvideo*+bestaudio/best` |
| 最佳兼容 | 兼容性最强（最高 1080p MP4） | `bestvideo[height<=1080][ext=mp4]+bestaudio[ext=m4a]/best[height<=1080]` |
| 最高 1080p | 限制最高 1080p | `bestvideo[height<=1080][ext=mp4]+bestaudio[ext=m4a]/best[height<=1080]` |
| 最高 4K | 限制最高 4K | `bestvideo[height<=2160]+bestaudio/best[height<=2160]` |
| 仅音频 | 只下载音频 | `bestaudio[ext=m4a]/bestaudio` |
| 最小体积 | 更小体积 | `best[height<=480]/worst` |
| 保留原始编码 | 保留原始编码组合 | `bestvideo+bestaudio/best` |
| HDR 优先 | 优先 HDR | `bestvideo[dynamic_range=HDR]+bestaudio/best` |
| 高帧率优先 | 优先 50fps+ | `bestvideo[fps>=50]+bestaudio/best` |
| 手动选择格式 | 手动选择格式 ID | 由表格选择/双击决定 |

**本次更新补充**
- “仅音频”预设在“直接下载”路径已可直接使用，不再需要先经过手动格式选择。

### 3.4 输出格式与手动格式

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 输出格式 | `mp4` | `mp4` / `mkv` / `m4a` / `mp3` / `opus` / `wav` / `flac` | 依据策略变化；音频策略时为音频格式 |
| 手动格式 | 未选择 | 来自格式表 | 仅在“手动选择格式”或双击格式表后生效 |

**补充说明**
- 视频类预设常用输出格式为 `mp4 / mkv`。
- 仅音频预设常用输出格式为 `m4a / mp3 / opus / wav / flac`。
- 若手动指定了带视频的 `format_id`，最终输出仍受所选流与后处理能力约束。

### 3.5 格式获取与筛选（仅 YouTube 模式）

| 筛选 | 默认值 | 说明 |
|------|--------|------|
| 仅 MP4 | 关闭 | 仅显示 mp4 容器 |
| 仅带音频 | 关闭 | 仅显示含音频流的格式 |
| 仅 60fps | 关闭 | 仅显示 fps ≥ 50 |
| 仅 4K+ | 关闭 | 仅显示分辨率 ≥ 2160p |
| 仅音频轨 | 关闭 | 仅显示音频流格式 |

| 排序 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 排序方式 | `quality_desc` | `quality_desc` / `quality_asc` / `size_desc` / `size_asc` | 画质或大小排序（仅对已解析格式有效） |

**格式表列**：`format_id` / `ext` / `resolution` / `fps` / `vcodec` / `acodec` / `protocol` / `filesize` / `dynamic_range` / `note`。

**补充说明**
- 表头当前为左对齐，便于快速浏览不同列名。
- “获取分辨率/格式”按钮已去掉立体阴影，视觉上与页面其他操作按钮更一致。
- 格式表支持横向滚动，适合查看 `note`、`dynamic_range`、`protocol` 等较长列。
- `note` 中常会出现“仅视频 / 仅音频 / 含音频 / 需合并 / HDR / fps”等组合信息，可作为手动选择的重要参考。
- 如果你只是想快速下载，通常不需要手动选择格式；优先使用预设即可。

> 提示：双击格式表行可快速选择手动格式。

### 3.6 音频导出 / 重命名 / 预览

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 音质 (k) | `192` | `128` / `192` / `256` / `320` | 仅在 mp3/opus/wav/flac 转码时生效 |
| 重命名 | 空 | 文本 | 留空则自动从网页/标题生成 |
| 命名预览 | 自动生成 | - | 预览最终输出文件名 |

**补充说明**
- 自定义文件名会经过校验：不能包含 Windows 非法字符，也不能以空格或点号结尾。
- “命名预览”会随着标题、输出格式、自定义文件名等内容变化实时刷新，便于在真正入队前先确认最终文件名效果。
- 如果留空不填，程序会优先尝试使用网页标题；如果标题也未获取成功，则回退到按预设生成的默认名称。
- 音质参数主要在需要音频转码时有意义；如果只是直接得到原始音频流，最终结果仍受源站音轨本身限制。

### 3.7 网络与高级参数

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 代理 | 空 | 任意协议 URL | 必须包含协议头，如 `http://` / `socks5://` |
| Cookies 模式 | `file` | `file` / `browser` | 与全局设置同步 |
| Browser Cookies | 空 | `chrome` / `edge` / `firefox` | 仅 browser 模式有效 |
| 高级参数 | 空 | 白名单参数 | 仅允许安全参数，详见“高级参数说明” |

**补充说明**
- 代理地址必须写成完整 URL；缺少协议头时，程序会直接拦截并提示格式无效。
- `browser` 模式下如果未选浏览器，程序会尽量回退到更安全的默认行为，避免生成无效任务。
- 高级参数并不是“任意命令行透传”，而是受白名单约束；像 `--format`、`--proxy`、`--cookies`、`--download-sections` 这类核心参数不允许覆盖。
- 如果你已经在页面中设置了代理、Cookies、区段或字幕等主参数，通常不需要再通过高级参数重复书写一次。

### 3.8 区段下载

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 区段下载 | 空 | `HH:MM:SS-MM:SS` / `MM:SS-MM:SS` | 例如 `00:01:00-00:03:30` |

**补充说明**
- 适合下载长视频中的一小段，例如直播回放切片、课程片段、素材试下。
- 区段格式书写错误时，任务会在入队前被拦截，而不是等到运行中才失败。
- 当前程序包含“区段下载 fallback”逻辑：如果站点原生区段下载不稳定，程序会尝试先全量下载，再在本地用 `ffmpeg` 裁剪出目标片段。
- 因为存在本地裁剪回退流程，区段下载任务在某些场景下会比普通任务更占用磁盘空间和耗时。

### 3.9 字幕设置

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 字幕模式 | `none` | `none` / `manual` / `auto` / `both` | 手动/自动/全部 |
| 字幕语言 | 空 | 由 yt-dlp 决定 | 如 `zh,en`；支持多语言 |
| 字幕格式 | 空 | 由 yt-dlp 决定 | 如 `srt` / `vtt` / `ass` |
| 写入外挂字幕 | 开启 | 开关 | 输出独立字幕文件 |
| 嵌入字幕 | 关闭 | 开关 | 将字幕封装到视频文件（mkv 更稳） |

**补充说明**
- `manual` 通常表示抓取普通人工字幕，`auto` 表示抓取自动生成字幕，`both` 表示两类都尝试。
- 多语言一般使用英文逗号分隔，例如 `zh,en,ja`；是否能成功命中，取决于站点真实提供了哪些字幕轨。
- `写入外挂字幕` 与 `嵌入字幕` 可以同时开启：一种是保留独立字幕文件，一种是写入容器内部。
- `嵌入字幕` 在 `mkv` 容器中通常兼容性更好；`mp4` 下能否稳定嵌入，还取决于源字幕格式与 `ffmpeg` 处理链。
- 如果源视频根本没有字幕，开启这些选项也不会凭空生成字幕；日志中通常会给出没有可用字幕的提示。

### 3.10 后处理（Post-Processing）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 嵌入封面 | 开启 | 将封面嵌入视频文件 |
| 嵌入元数据 | 开启 | 写入标题/作者/描述等元数据 |
| 写入缩略图文件 | 关闭 | 另存封面图片文件 |
| 写入信息 JSON | 关闭 | 保存元数据 JSON 文件 |
| 写入描述 | 关闭 | 保存描述文本 |
| 章节写入 | 关闭 | 写入章节信息 |
| 保留中间视频 | 关闭 | 合并后不删除临时流文件 |
| H.264 兼容模式 | 关闭 | 触发转码为 H.264（会耗时） |
| SponsorBlock | 关闭 | 删除指定分类片段（默认 `sponsor`，可自定义） |
| 启用 PO Token | 关闭 | 为 YouTube 注入 PO Token（需 Node.js） |

**补充说明**
- `嵌入封面` 和 `嵌入元数据` 默认开启，适合希望下载结果更接近“整理后的成品文件”。
- `保留中间视频` 常用于排查合并失败、转码失败或想保留原始流文件的场景；平时关闭可减少磁盘占用。
- `H.264 兼容模式` 会引入转码步骤，通常更耗时、更吃 CPU，也更可能生成额外临时文件；只有在设备兼容性要求高时才建议开启。
- `SponsorBlock` 输入框默认是 `sponsor`，你也可以按 `yt-dlp` 支持的分类规则填写多个分类，但建议先小样本测试效果。
- `启用 PO Token` 只对部分 YouTube 抽取场景有帮助，不是所有视频都必须开启；如果普通公开视频能正常下，通常没必要默认常开。

### 3.11 下载控制

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 重试次数 | `3` | `0 - 10` | `0` 表示不重试 |
| 并发数 | `1` | `1 - 10` | 单视频页可配置并发，影响队列调度 |
| 限速 (MB/s) | `2` | `0 - 100` | `0` 表示不限速（单任务） |

**补充说明**
- 单视频页的“并发数”并不只是当前这一条 URL 的内部子线程概念，它会影响下载管理器对队列任务的并发调度上限。
- `重试次数` 更适合应对网络抖动、站点临时异常、连接中断等问题；如果是参数填写错误或认证状态错误，重复重试通常没有意义。
- `限速 = 0` 表示不限速，但并不一定意味着整体效率最高；磁盘、网络和站点风控都可能成为新的瓶颈。
- 当你在队列里同时堆积多条任务时，建议结合并发数和限速一起调节，而不是单独把某一项拉满。

### 3.12 单视频页的推荐使用顺序

如果你是第一次接触单视频页，推荐按下面顺序使用：

1. 先确认当前是 `YouTube` 还是 `Generic` 模式。
2. 粘贴 URL，先观察“视频信息卡”是否能正常解析。
3. 如果只是普通下载，优先直接选预设，不必一开始就进入手动格式。
4. 如果你明确要特定分辨率/编码，再点击“获取分辨率/格式”，用筛选缩小范围。
5. 先看“命名预览”和“下载摘要”，确认输出格式、后处理、字幕与命名是否正确。
6. 最后再决定是“直接下载”还是“添加到队列”。

这样做的好处是：可以在真正启动下载前，把大部分参数问题提前暴露出来，减少入队后才发现配置错误的情况。

## 4. 批量页参数（播放列表 / 频道）

### 4.1 URL / 解析 / 筛选与表格

**按钮与动作**：
- **解析批量条目**：解析播放列表/频道条目。
- **全选 / 全不选 / 仅保留可用**：批量勾选条目。
- **添加选中到队列**：将勾选条目加入下载队列。

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 来源类型 | `auto` | `auto` / `playlist` / `channel` | 用于提示解析方式 |
| 隐藏不可用 | 关闭 | 开启 / 关闭 | 隐藏不可下载条目 |
| 仅 Shorts | 关闭 | 开启 / 关闭 | 仅显示 Shorts |

**表格列**：选择 / 序号 / 标题 / 频道 / 时长 / 观看 / 上传日期 / 可用性 / Shorts / URL。

**补充说明**
- 解析成功后，批量页顶部会同步刷新“来源标题 / 来源信息 / 总条目统计”，方便先判断这是不是你想要的播放列表或频道。
- “仅保留可用”适合在大列表里快速剔除不可访问条目，减少后续入队噪音。
- `仅 Shorts` 更适合做短视频素材批量整理；如果不开启，则 Shorts 与普通视频会一起显示。
- 批量页的勾选状态会直接影响“添加选中任务”的入队范围，因此建议先筛选、再勾选、最后统一入队。

### 4.2 下载策略与输出

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 下载策略 | `best_compat` | best_quality / best_compat / max_1080p / max_4k / audio_only / min_size | 批量统一策略 |
| 输出格式 | `mp4` | `mp4` / `mkv` / `webm`（音频策略时为 `m4a/mp3/opus/wav/flac`） | 自动根据策略切换 |
| 音频质量 | `192` | `128` / `192` / `256` / `320` | 仅音频策略有效 |
| 自定义前缀 | 空 | 文本 | 批量生成文件名时作为前缀 |

**补充说明**
- 批量页的这些参数会在“添加选中任务”时整体复制到当前勾选的所有任务，因此建议先全部确认好再统一入队。
- `音频质量` 主要用于音频转码场景；如果最终拿到的是原始音轨，不一定会严格等于你填写的码率值。
- `自定义前缀` 适合给整批任务增加统一主题标识，例如课程名、项目代号、日期批次等。
- `manual` 批量手动策略启用后，真正生效的下载表达式将优先由手动规则决定，而不是普通预设单选项本身。
- `webm` 输出已支持批量页。
- `webm` 输出与 `H.264 兼容模式` 不应同时启用；界面会阻止明显冲突的组合入队。

### 4.3 批量手动策略

**入口**
- 勾选 `启用手动`
- 点击 `手动设置`

**基本流程**
1. 选择一个样本视频 URL，或直接从当前已选条目中取样本。
2. 点击“获取格式”，读取该样本视频的可用格式集合。
3. 配置预设1。
4. 如有需要，可启用并配置预设2。
5. 如有需要，可启用“最终兜底链”。
6. 保存后再执行批量入队。

**当前支持的规则维度**
- 高度
- 编码偏好：`h264 / av1 / vp9`
- 视频流容器偏好：`mp4 / webm`
- 音频模式：`default / no_audio`

**当前支持的层级**
- 预设1
- 预设2（可选）
- 最终兜底链（可选）

**最终兜底链顺序说明**
1. 先尝试与目标高度相同的可用格式。
2. 若没有命中，则继续尝试更高的可用格式。
3. 若仍没有命中，则继续尝试更低的可用格式。
4. 最后回退到通用的 `bestvideo/best`。

**使用说明**
- 这套批量手动策略是规则匹配，不是对每一条任务预先锁定某个固定 `format_id`。
- 实际命中结果仍以 `yt-dlp` 在该视频上的可用格式为准。
- 手动策略启用后，任务摘要会显示为 `manual` 路径，而不是原来的普通预设名。

**当前限制**
- 音频高级筛选目前仍以 `default / no_audio` 为主，尚未开放更细的音频码率/音频格式规则选择。
- UI 当前不会逐条提前展示“最终命中的是哪一个具体格式 ID”；真实命中结果以运行时下载日志为准。
- 若样本 URL 被修改，需要重新获取样本格式后再保存策略。

**详细参数说明**

1. `样本 URL`
   - 用途：只用于读取一个代表性视频的可用格式，帮助你构造规则；它不是“把这一条视频固定照搬到整批任务”。
   - 推荐：优先选择与目标批量内容风格接近、格式较完整、能正常播放的一条视频作为样本。
   - 影响：样本拉取成功后，界面中的高度、编码、视频流容器可选项会按该样本的真实格式刷新。
   - 注意：修改样本 URL 后，旧的样本格式会立即失效，必须重新点击“获取格式”。

2. `取已选样本`
   - 用途：从当前批量列表中已勾选/已选中的条目里快速取一条 URL 填入样本框。
   - 注意：这个按钮只负责带入 URL，不会自动获取格式；填入后仍需点击“获取格式”。

3. `获取格式`
   - 用途：调用当前样本视频，读取其格式集合，并作为手动规则 UI 的候选数据源。
   - 结果：成功后，下拉框会更新为该样本实际存在的高度、编码、视频流容器。
   - 失败处理：如果样本视频不可访问、网络异常、站点风控或 `yt-dlp` 无法解析，手动策略无法保存。

4. `预设1`
   - 作用：这是第一层规则，也是启用手动策略后的主规则。
   - 必填性：只要勾选“启用手动”，预设1就必须有效。
   - 校验规则：预设1至少要指定一个视频条件，即“高度 / 编码 / 视频流容器”三者中至少选一个；全部保持“任意”时不能保存。
   - 推荐：把预设1设成你最希望命中的目标规格，例如 `1080p + h264 + mp4 + 默认合并音频`。

5. `预设2`
   - 作用：第二层规则，仅在预设1未命中时继续尝试。
   - 是否必须：不是必须；只有勾选“启用预设2”时才参与匹配。
   - 校验规则：启用预设2后，它也必须至少包含一个视频条件。
   - 推荐：预设2应比预设1更宽松，而不是更苛刻。常见做法是降低高度，或放宽编码偏好。

6. `启用最终兜底链`
   - 作用：在预设1、预设2都没有命中时，再追加一条更宽的收尾规则。
   - 基准来源：如果启用了预设2，兜底链以预设2为基础继续扩展；如果没有启用预设2，则以预设1为基础。
   - 适用场景：当你的批量来源比较杂，不同视频的可用格式差异很大时，建议开启。
   - 效果边界：兜底链只能放宽“高度命中方式”，不会把“视频流容器偏好”改成另一个容器。

7. `高度`
   - 含义：目标视频流的高度条件，常见如 `2160 / 1440 / 1080 / 720 / 480`。
   - 在预设1/预设2中的行为：属于“目标高度”条件；如果设置了 `1080`，该层会优先尝试 `height=1080`。
   - 在兜底链中的行为：如果开启兜底，会继续按“同高度 -> 更高 -> 更低 -> 通用 bestvideo/best”的顺序扩展。
   - 设为“任意”：表示这一层不限制高度。

8. `编码`
   - 当前支持：`h264 / av1 / vp9`。
   - 语义：这里是“编码偏好”，不是“绝对只允许这一种编码”。
   - 当前实现：如果你选择了 `vp9`，会先尝试 `vp9`，再按内部顺序尝试其他编码；因此它更接近“优先顺序”而不是“硬锁定”。
   - 选择建议：
     - `h264`：兼容性最好，适合通用设备和后续编辑。
     - `vp9`：常见于 `webm` 路径，适合希望保持较新网页流规格时使用。
     - `av1`：体积和画质比通常更优，但并非所有视频都提供。

9. `视频流容器`
   - 当前支持：`mp4 / webm`。
   - 语义：这是“源视频流容器偏好”，不是批量页顶部那个“最终输出格式”。
   - 重要区别：
     - 这里控制的是 yt-dlp 在挑选视频流时优先找 `mp4` 还是 `webm` 视频流。
     - 批量页主界面的“输出格式”控制的是下载完成后的最终封装方向，例如 `mp4 / mkv / webm`。
   - 推荐：
     - 想走兼容性优先时，通常选 `mp4`。
     - 想尽量贴近网页侧 `vp9/webm` 资源时，可选 `webm`。

10. `音频`
   - 当前支持：`默认合并` 与 `不下载音频`。
   - `默认合并`：为该层视频规则追加 `+bestaudio`，适合正常的视频下载。
   - `不下载音频`：该层只下载视频流，不再拼接音频。
   - 当前限制：还不支持在这里手动指定音频扩展名、音频码率或单独音频格式 ID。

**规则命中顺序说明**
- 整体顺序始终是：`预设1 -> 预设2（如果启用） -> 最终兜底链（如果启用）`。
- 每个预设层内部会根据“编码偏好”生成一个有顺序的候选链，而不是只生成单个表达式。
- 如果开启了最终兜底链：
  - 有目标高度时：按 `同高度 -> 更高可用 -> 更低可用 -> bestvideo/best`。
  - 没有目标高度时：直接从当前容器/编码偏好可用链开始，再回退到 `bestvideo/best`。
- 是否真正命中其中哪一段，最终仍由运行时 `yt-dlp` 对该视频的真实可用格式决定。

**推荐配置方式**

1. `通用兼容型`
   - 预设1：`1080 + h264 + mp4 + 默认合并`
   - 预设2：`720 + h264 + mp4 + 默认合并`
   - 最终兜底链：开启
   - 适合：播放列表来源较杂，希望优先成功率和兼容性。

2. `网页流优先型`
   - 批量页输出格式：`webm`
   - 预设1：`1080 + vp9 + webm + 默认合并`
   - 预设2：`720 + webm + 默认合并`
   - 最终兜底链：开启
   - 适合：希望尽量保持 `webm / vp9` 路线。

3. `静音素材型`
   - 预设1：`1080 + mp4 + 不下载音频`
   - 预设2：`720 + mp4 + 不下载音频`
   - 最终兜底链：按需要开启
   - 适合：只需要画面、不需要音轨的素材整理或二次处理。

**保存与入队注意事项**
- 手动策略保存成功后，批量页摘要会显示为 `manual` 路径。
- 只勾选“启用手动”但没有保存有效策略时，批量入队会被阻止。
- 样本只是“构造规则的参考来源”，不是保证整批条目都具备相同格式。
- 如果你的规则过窄且又没有启用预设2或兜底链，不同视频在运行时可能出现“无可用格式命中”的情况。
- `webm` 输出与 `H.264 兼容模式` 不应同时启用；前者偏向 `webm` 路线，后者则要求 H.264 兼容处理。

### 4.4 区段下载

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 区段下载 | 空 | `HH:MM:SS-MM:SS` / `MM:SS-MM:SS` | 例如 `00:01:00-00:03:30` |

### 4.5 字幕与后处理

与单视频页一致：字幕模式 / 语言 / 格式、外挂 / 内嵌、封面 / 元数据 / 描述 / 章节、H.264 兼容、SponsorBlock、PO Token。默认值与单视频页一致。

**补充说明**
- 批量页这里的字幕和后处理选项，会整体作用于当前选中的所有条目，因此一旦配置过重，可能让整批任务都变慢。
- 如果你只是先验证播放列表是否能稳定下载，建议第一轮先关闭复杂后处理（如 H.264 转码、SponsorBlock、多字幕嵌入），确认稳定后再逐步加功能。
- 对于条目数量较多的批量任务，`写入信息 JSON / 描述 / 缩略图` 会明显增加额外文件数量，适合需要做归档整理时再开启。

### 4.6 网络与高级参数

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 代理 | 空 | 任意协议 URL | 同单视频页 |
| Cookies 模式 | `file` | `file` / `browser` | 与全局设置同步 |
| Browser Cookies | 空 | `chrome` / `edge` / `firefox` | 仅 browser 模式有效 |
| 高级参数 | 空 | 白名单参数 | 仅允许安全参数 |

**补充说明**
- 批量页的网络参数一旦入队，会被复制到每条批量任务中，因此这里更适合填“整批统一参数”，不适合做逐条差异化配置。
- 如果你怀疑整批链接都受同一网络环境影响，优先在批量页统一设置代理或 Browser Cookies，而不是等任务失败后逐条补救。
- 批量高级参数仍然受白名单控制；它适合做统一微调，不适合覆盖主界面已经提供的显式字段。

### 4.7 字幕

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 字幕模式 | `none` | `none` / `manual` / `auto` / `both` | 手动 / 自动 / 全部 |
| 字幕语言 | 空 | 由 yt-dlp 决定 | 如 `zh,en`；支持多语言 |
| 字幕格式 | 空 | 由 yt-dlp 决定 | 如 `srt` / `vtt` / `ass` |
| 写入外挂字幕 | 开启 | 开关 | 输出独立字幕文件 |
| 嵌入字幕 | 关闭 | 开关 | 将字幕封装到视频文件（mkv 更稳） |

**补充说明**
- 批量字幕参数与单视频页逻辑一致，但因为对象是一整批条目，建议先保证语言填写比较宽松，否则容易出现“部分条目有字幕、部分条目没有字幕”的混合结果。
- 如果你要长期保存课程、讲座、访谈类内容，通常建议至少开启外挂字幕，便于后期检索和二次处理。
- 如果你更看重“单文件成品化”，可以考虑再开启嵌入字幕，但要预期封装与后处理时间会增加。

### 4.8 节流（Throttle）

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 请求间隔 (s) | `5` | `0 - 60` | `--sleep-interval` |
| 最大间隔 (s) | `10` | `0 - 120` | `--max-sleep-interval` |
| API 间隔 (s) | `1` | `0 - 30` | `--sleep-requests` |
| 重试间隔 (s) | `10` | `0 - 300` | `--retry-sleep http:` |

**补充说明**
- 节流参数主要用于降低批量请求节奏，减少触发站点频率限制的概率。
- 当你遇到播放列表解析成功、但批量下载阶段经常 403、限流或中断时，可以优先适度提高这些间隔值，而不是单纯增加重试次数。
- 如果网络环境已经很稳定、批量量级也不大，可以先保持默认值；过度节流会明显拉长整体下载时间。

### 4.9 下载控制

| 参数 | 默认值 | 取值范围 | 说明 |
|------|--------|----------|------|
| 重试次数 | `3` | `0 - 10` | `0` 表示不重试 |
| 并发数 | `1` | `1 - 10` | 批量并发数 |
| 限速 (MB/s) | `2` | `0 - 100` | `0` 表示不限速 |

**补充说明**
- 批量任务数量大时，真正决定整体速度的往往不是单条下载速度，而是并发、节流、网络稳定性与站点风控的平衡。
- 若你下载的是长播放列表，建议先用较保守配置跑前 5~10 条，确认没有大面积失败后，再适度放宽并发或限速。
- 如果你是代理环境或跨境网络环境，过高并发反而更容易触发连接失败、下载中断或超时。

## 7. 认证与安全

### 7.1 Cookies 文件

| 项目 | 说明 |
|------|------|
| 默认文件名 | `www.youtube.com_cookies.txt` |
| 存放路径 | 程序目录（默认）或配置路径 |
| 作用 | 解决 403 / 年龄限制 / 地区限制 |

**使用方法**：
1. 浏览器登录 YouTube。
2. 使用扩展导出 `www.youtube.com` 的 cookies（Netscape 格式）。
3. 将文件放到程序目录或按配置路径保存。

**补充说明**
- 如果程序检测不到本地 Cookies 文件，通常不会阻止普通公开视频下载，但访问受限内容时成功率会明显下降。
- 当你已经启用了 Browser Cookies 时，本地 cookies 文件不是强制项。
- 认证状态中心会展示文件是否存在、最近检查结果、最近错误分类以及诊断建议。

### 7.2 Browser Cookies

- 支持浏览器：Chrome / Edge / Firefox。
- 在设置页或顶部选择 `browser` 并指定浏览器名称即可生效。
- Browser Cookies 适合不想手动导出 cookies 文件，或 cookies 文件经常过期的场景。

### 7.3 PO Token

- 需要 Node.js（建议 ≥18；当前程序的状态检测与提示也以较新的 Node.js 环境为目标）。
- 程序会自动初始化 Token；状态显示在“认证状态中心”。
- 当启用 PO Token 选项时，会在 yt-dlp 里注入 `extractor-args`。
- 如果当前发布包未附带 PO Token 工具，状态区会显示“未启用 / Disabled”，这属于正常降级，不代表程序报错。
- 对于大多数公开视频，未启用 PO Token 也可以正常使用；只有在遇到特定风控或抽取限制时，才建议进一步排查 PO Token。

### 7.4 认证状态中心建议查看什么

进入“认证状态中心”后，建议重点关注以下内容：

1. `Cookies Mode`：确认当前到底走 `file` 还是 `browser`。
2. `Cookies File Path`：确认路径是否指向正确文件。
3. `Last Check`：判断最近一次 Cookies 检查是否成功。
4. `Last Error Category`：确认是文件缺失、登录失效、站点风控还是其他问题。
5. `PO Token Status`：判断当前是 `ready / disabled / no_node / error` 哪一类状态。
6. `Suggested Actions`：程序会根据最近诊断给出下一步建议，优先级通常高于盲目重复尝试。

## 8. 组件中心

| 项目 | 说明 |
|------|------|
| yt-dlp | 显示路径 / 版本 / 状态，支持一键更新 |
| ffmpeg | 显示路径 / 版本 / 状态（不提供更新按钮） |
| deno | 显示路径 / 版本 / 状态（不提供更新按钮） |
| 导出诊断 | 导出组件诊断 JSON |

**补充说明**
- 组件中心里的“更新”按钮当前是统一更新入口，会一起处理 `yt-dlp / ffmpeg / deno`，而不是只更新其中一个。
- 如果你只缺少 `yt-dlp` 或 `ffmpeg`，也可以直接使用该入口，让程序自动补齐。
- YCB 只识别程序目录内的 `yt-dlp / ffmpeg / deno`，不会读取系统 `PATH` 里的同名程序。
- 因此，如果你的电脑已经在系统环境变量里装过这些工具，但程序目录内没有对应文件，组件中心仍会显示缺失，运行状态也会按“缺少组件”处理。
- 安装器里的组件检测也遵循同一规则：只检查程序目录内是否已有组件，不会把系统 `PATH` 中的现成程序算作已安装。
- `deno` 主要影响 Hook 与部分扩展能力；没有它不一定影响普通下载，但会限制扩展脚本功能。
- 导出诊断适合在排查“版本不对 / 路径异常 / 组件不可用”时留存现场信息。

## 11. 高级参数白名单（yt-dlp）

仅允许以下安全参数（其余将被拒绝）：

- 无值参数：
  - `--no-part`
  - `--no-continue`
  - `--force-overwrites`
  - `--ignore-errors`
  - `--abort-on-error`
  - `--force-ipv4`
  - `--no-check-certificate`

- 需要值的参数：
  - `--concurrent-fragments` / `-N`
  - `--fragment-retries`
  - `--extractor-retries`
  - `--file-access-retries`
  - `--socket-timeout`
  - `--http-chunk-size`
  - `--downloader`
  - `--downloader-args`
  - `--compat-options`

**使用建议**
- 适合放在这里的通常是“下载层微调参数”，例如分片并发、超时、兼容选项、下载器选择等。
- 不适合放在这里的，是那些已经由界面主参数直接管理的选项，例如格式、代理、Cookies、区段、字幕、输出路径等。
- 如果你不确定某个参数是否安全，优先不要写；先用界面现有开关和输入项完成配置，往往更稳。
- 当高级参数与主界面参数语义重叠时，程序会优先保护主界面参数，避免被覆盖后导致任务行为不可预期。

**示例**
- 想提高分片并发：可尝试 `--concurrent-fragments 4`
- 想放宽某些兼容行为：可尝试 `--compat-options no-youtube-unavailable-videos`
- 想调大网络超时：可尝试 `--socket-timeout 30`

> 注意：与应用内部参数冲突的选项将被拒绝（如 `--format` / `--proxy` / `--cookies` / `--download-sections` 等）。

## 12. Hook 扩展、日志与诊断

### 12.1 Hook 扩展

- 程序支持基于 `deno` 的 Hook 扩展能力，用于在任务开始、完成、失败等时机执行额外逻辑。
- Hook 更适合做自动归档、通知、二次记录、发布流程衔接等高级扩展，不属于普通下载必须项。
- 如果当前环境没有 `deno` 或发布包未附带相关 Hook 资源，程序通常会退化为“主功能可用、扩展功能不可用”。

### 12.2 日志阅读建议

- 下载日志中：
  - `INFO` 主要用于普通状态流转；
  - `WARN / WARNING` 常表示存在风险或建议调整参数；
  - `ERROR` 常表示任务失败或关键步骤异常；
  - `[摘要]` 行通常用于展示命令、参数、输出目录等概览信息。
- 媒体工具日志与下载队列日志是分离的；排查时先确认你看的是哪一种任务系统。
- 如果你在日志里看到大量重复重试，优先回头检查认证、代理、格式策略，而不是单纯继续增加重试次数。

### 12.3 建议的排障顺序

1. **先看组件中心**：确认 `yt-dlp / ffmpeg / deno` 是否存在且版本正常。
2. **再看认证状态中心**：确认 Cookies、Browser Cookies、PO Token 是否可用。
3. **再看运行状态中心**：判断最近一次问题属于组件、认证、数据库还是运行期问题。
4. **最后看具体日志**：回到下载队列或媒体工具日志，定位触发时机、摘要和原始错误输出。

### 12.4 文档与实际行为的关系

- 本手册基于当前代码行为整理，适合作为使用与排障说明书。
- 如果将来 UI 文案、按钮位置或默认值有小幅调整，应优先以程序界面、实时日志和状态中心提示为准。
- 对于复杂功能（如批量手动策略、区段 fallback、PO Token、Hook），建议始终先小范围测试，再投入大批量使用。
