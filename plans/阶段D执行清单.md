# 阶段 D 执行清单

## 1. 阶段定位

阶段 D 面向 **播放列表与频道能力增强**，目标是在阶段 C 已完成的单视频专业工作台基础上，补齐 YouTube 批量内容解析、预览、筛选、批量入队与最小验证闭环。

本阶段不提前引入阶段 E/F 的订阅同步、历史数据库重构或归档系统复杂度，优先完成：

- Playlist 批量解析
- Channel / Uploads 批量解析
- 批量条目预览与筛选
- 批量任务入队与基础去重
- 阶段 D 冒烟验证与结项记录

---

## 2. 执行原则

- 不破坏现有单视频能力 [`ui/pages/single_video.py`](ui/pages/single_video.py)
- 批量能力优先复用 [`core/youtube_metadata.py`](core/youtube_metadata.py) 与 [`core/ytdlp_builder.py`](core/ytdlp_builder.py)
- 先实现解析、展示、筛选、入队主路径，再考虑复杂同步场景
- 所有新能力必须围绕 YouTube 批量下载展开，不扩展回多站点架构
- 阶段 D 完成后必须执行真实样本冒烟并形成文档

---

## 3. 执行清单

### D1. 批量领域模型设计

- [ ] 在 [`core/youtube_models.py`](core/youtube_models.py) 新增播放列表与频道解析结果模型
- [ ] 为批量条目定义统一字段：`video_id`、`title`、`url`、`duration`、`availability`、`channel`、`is_shorts`
- [ ] 为批量来源定义统一字段：`source_type`、`source_id`、`source_title`、`entry_count`
- [ ] 预留后续阶段 E/F 需要的基础字段，例如 `channel_id`、`uploader_id`
- [ ] 保证新模型可直接映射到现有下载任务结构 [`YouTubeTaskRecord`](core/youtube_models.py:69)

### D2. 元数据服务扩展

- [ ] 在 [`core/youtube_metadata.py`](core/youtube_metadata.py) 增加播放列表解析接口
- [ ] 在 [`core/youtube_metadata.py`](core/youtube_metadata.py) 增加频道 Uploads / Videos 解析接口
- [ ] 统一返回批量结果对象，避免 Playlist 与 Channel 分别维护两套结构
- [ ] 处理空结果、不可访问内容、部分视频不可用等错误场景
- [ ] 保留 cookies 兜底与错误识别逻辑，保持与当前单视频链路一致

### D3. 批量页面与主界面接入

- [ ] 新增阶段 D 的批量下载页面模块，建议放入 [`ui/pages/`](ui/pages)
- [ ] 在主界面中接入新页面，不影响现有单视频页与历史页
- [ ] 页面输入区支持：Playlist URL、Channel URL、Uploads URL
- [ ] 页面摘要区显示来源标题、来源类型、总条目数、当前筛选结果数
- [ ] 明确错误提示文案，避免批量页面出现静默失败

### D4. 批量条目预览与筛选

- [ ] 在批量页面增加条目表格，展示标题、视频 ID、时长、可用性、频道等字段
- [ ] 支持单条勾选、全选、全不选、反选
- [ ] 支持基础筛选：仅可下载、排除 Shorts、标题关键词过滤
- [ ] 支持限制导入数量，例如前 N 条
- [ ] 实时更新批量摘要，展示总条目数、已选条目数、预计入队数

### D5. 批量下载配置复用

- [ ] 在批量页面复用阶段 C 已有下载策略体系
- [ ] 支持统一设置：最佳兼容、最高 1080p、仅音频、H.264 兼容模式等
- [ ] 支持统一输出格式与后处理选项
- [ ] 明确哪些配置对批量场景生效，避免单视频特有交互直接照搬造成歧义
- [ ] 确保配置摘要可读，便于用户在批量入队前确认设置

### D6. 批量入队与去重

- [ ] 将批量选中条目转换为现有下载任务对象并接入队列管理器
- [ ] 批次内按 `video_id` 或标准化 URL 去重
- [ ] 对与已有任务重复的条目给出提示或跳过策略
- [ ] 确保失败条目不会阻断整批入队
- [ ] 为后续历史数据库去重保留扩展点，但本阶段不做数据库重构

### D7. 批量场景交互收口

- [ ] 对空输入、非法链接、无条目返回提供清晰提示
- [ ] 对部分不可访问视频提供 warning 日志与 UI 提示
- [ ] 对批量筛选后无结果提供空状态提示
- [ ] 对批量入队结果提供汇总反馈，例如成功数、跳过数、失败数
- [ ] 检查与现有日志系统、状态栏、任务列表的文案一致性

### D8. 阶段 D 冒烟验证

- [ ] 验证程序启动正常，不影响阶段 C 单视频工作台
- [ ] 用一个真实 Playlist 链接验证批量解析与条目展示
- [ ] 用一个真实 Channel / Uploads 链接验证批量解析与条目展示
- [ ] 验证至少一轮批量筛选与部分勾选入队
- [ ] 验证至少一种批量下载策略，如“最佳兼容”或“仅音频”
- [ ] 验证典型失败场景：空结果、不可访问视频、重复条目、筛选后无结果

### D9. 阶段 D 文档收口

- [ ] 更新阶段 D 当前进展文档，记录每批功能完成状态
- [ ] 记录真实样本验证结果与遗留风险
- [ ] 编写正式阶段 D 结项与冒烟记录文档
- [ ] 明确哪些能力留待阶段 E/F 继续推进

---

## 4. 推荐实施顺序

1. 完成 [`core/youtube_models.py`](core/youtube_models.py) 的批量模型设计
2. 完成 [`core/youtube_metadata.py`](core/youtube_metadata.py) 的 Playlist / Channel 解析能力
3. 新建批量页面并接入主界面
4. 接入批量表格、筛选、选择与摘要
5. 接入批量下载配置复用
6. 接入批量入队与去重逻辑
7. 执行真实链路冒烟与问题修复
8. 编写阶段 D 结项文档

---

## 5. 阶段 D 完成标准

满足以下条件后，阶段 D 才视为完成：

- Playlist 与 Channel 两类来源均可解析
- 批量条目可预览、筛选、选择并入队
- 至少一条批量下载主路径完成真实样本验证
- 不破坏现有单视频页 [`ui/pages/single_video.py`](ui/pages/single_video.py)
- 已完成阶段 D 冒烟验证并形成正式记录文档
