# 能力覆盖清单：yt-dlp / FFmpeg

## 1. 文档目的

本文档用于长期跟踪当前项目对 yt-dlp 与 FFmpeg 官方能力的接入情况。

它不是阶段性方案，而是能力盘点表。后续每完成一项功能，都应更新本表。

接入方式定义：

- `核心 UI`：主流程中直接展示
- `高级 UI`：在高级区、展开区或独立设置页展示
- `模板`：通过预设、策略或工作流模板接入
- `透传`：通过高级参数直接覆盖
- `脚本`：通过 Deno Hook 或外部脚本覆盖

状态定义：

- `未开始`
- `进行中`
- `已完成`
- `透传可用`
- `不计划直接 UI 化`

## 4. 通过透传与脚本覆盖的能力

以下能力不要求第一时间做独立 UI，但要保证最终可通过透传或脚本覆盖：

### yt-dlp

- 自定义 extractor args
- 自定义请求头
- 自定义 postprocessor args
- 更细粒度的站点特化参数
- 归档相关高级选项
- 评论导出、播放列表高级字段等低频能力

### FFmpeg

- 自定义滤镜链
- 自定义 `map`
- 自定义 metadata / disposition
- 自定义编码参数组合
- 批量脚本化工作流

## 6. 推荐维护方式

每轮开发完成后：

1. 先更新本表中的“当前状态”。
2. 再更新对应阶段结项文档。
3. 若某能力决定不做独立 UI，必须明确写成：
   - `透传可用`
   - 或 `不计划直接 UI 化`

这样可以防止后续再次把“暂不 UI 化”误判成“尚未支持”。
