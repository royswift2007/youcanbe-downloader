# 阶段2执行清单

## 1. 阶段定位

阶段 2 只做一件核心事情：把字幕系统做完整。

当前项目虽然已经有基础字幕逻辑，但仍然过于简单，不能满足专业使用场景。

本阶段的任务是把字幕从“单一开关”提升为“可控系统”，但仍然只围绕下载流程，不进入本地字幕处理工作台。

---

## 2. 本阶段完成标准

- 支持手工字幕
- 支持自动字幕
- 支持两者都要
- 支持多语言 / 语言文本输入
- 支持字幕格式选择
- 支持外挂、内嵌、两者并存
- 批量页与单视频页的字幕配置一致

---

## 3. 本阶段不做什么

- 不做字幕预览器
- 不做字幕轨复杂管理
- 不做本地 FFmpeg 字幕烧录工作台
- 不做字幕文件编辑器

## 7. 详细开发步骤

### 7.1 扩展 profile

1. 在 `YouTubeDownloadProfile` 中加入字幕系统字段。
2. 保留旧字段兼容逻辑。
3. 若已有 `sub_lang`，可在过渡期保留兼容映射。

### 7.2 扩展 builder

根据字幕模式输出对应参数：

- `manual` -> `--write-subs`
- `auto` -> `--write-auto-subs`
- `both` -> 两者都输出

同时处理：

- `--sub-langs`
- `--sub-format`
- `--embed-subs`

### 7.3 改造单视频页 UI

单视频页新增字幕区域，至少包括：

- 字幕模式
- 语言输入
- 字幕格式
- 输出方式

建议顺序：

1. 先做模式
2. 再做语言
3. 再做格式
4. 再做输出方式

### 7.4 改造批量页 UI

批量页采用统一字幕设置，不做每个条目单独字幕策略。

至少包括：

- 模式
- 语言
- 格式
- 外挂/内嵌

### 7.5 容器兼容说明

UI 中必须加说明：

- `mkv` 对字幕兼容更好
- `mp4` 下不要默认承诺所有字幕都能稳定内嵌
