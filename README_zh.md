# YCB（YouCanBe Downloader）

一个面向 YouTube 的桌面下载与媒体处理工具，基于 `yt-dlp` + `ffmpeg` +（可选）`deno`。提供单视频下载、批量下载（播放列表/频道）、下载队列、历史记录、媒体工具、组件中心、认证中心、运行状态中心，以及 Hook 扩展能力。

> PO Token 功能依赖本地 `Node.js v18+`。若只使用普通下载、Cookies 或 Browser Cookies，可不安装 Node.js。

- **界面语言**：中文 / English
- **运行方式**：源码运行（Python）或搭配内置组件运行
- **适用平台**：Windows（当前仓库提供 `.exe` 组件）

> English version: [`README.md`](README.md)
> 使用手册：[`usage_intro.md`](usage_intro.md)


> [!IMPORTANT]
> 首次运行前/首次运行时，请先进入 **设置 → 组件中心**，点击“**组件更新（yt-dlp/ffmpeg/deno）**”。
> 程序依赖这些组件完成下载与处理；如果本地还没有组件，请先更新再使用。

## ✨ 功能总览

- **单视频下载**：支持 YouTube 与 Generic URL（非 YouTube 通用下载）
- **批量下载**：支持播放列表/频道，批量筛选/勾选
- **批量手动策略**：支持样本视频、预设1/预设2、最终兜底链
- **下载策略**：最佳画质 / 最佳兼容 / 最高 1080p / 最高 4K / 仅音频 / 最小体积 / HDR 优先 / 高帧率优先 / 手动格式
- **格式表与筛选**：解析并筛选 YouTube 全部格式组合
- **队列管理**：批量开始 / 开始 / 重试 / 停止 / 删除 / 清除完成
- **历史记录**：可查看/刷新/清空历史列表
- **媒体工具**：remux、抽音频、裁剪、拼接、烧录字幕、缩放、裁切、旋转、水印、响度归一化
- **认证中心**：Cookies / Browser Cookies / PO Token 管理
- **组件中心**：yt-dlp / ffmpeg / deno 版本检测
- **运行状态中心**：队列/数据库/异常汇总
- **Hook 扩展**：支持自定义脚本扩展（见 `core/hooks.py`）

## 📦 运行方式

### 方式 A：源码运行（推荐开发者）

1. 安装 Python 3.10+（建议 3.10/3.11）
2. 安装依赖：

```bash
pip install -r requirements.txt
```

3. 运行：

```bash
python YCB.pyw
```

> Windows 下双击 `YCB.pyw` 可无控制台运行。

### 方式 B：搭配内置组件运行（面向普通用户）

确保以下组件位于程序目录：

- `yt-dlp.exe`
- `ffmpeg.exe`
- `deno.exe`（可选）

程序会自动检测并加载这些组件。

> 首次运行时，程序目录可能没有 `yt-dlp.exe / ffmpeg.exe / deno.exe`。
> 请进入 **设置 → 组件中心**，点击“**组件更新（yt-dlp/ffmpeg/deno）**”按钮，程序会自动下载并安装到根目录，下载过程会显示进度。

## 🧭 核心使用流程

### 1）单视频下载（YouTube / Generic）

1. 选择模式：
   - **YouTube 模式**：仅支持 YouTube 链接，提供格式解析
   - **Generic 模式**：支持任意 http/https 链接（无格式列表）
2. 输入链接
3. 选择操作：
   - **获取分辨率/格式**（仅 YouTube）
   - **直接下载**（自动策略；“仅音频”可直接按音频策略启动）
   - **添加到队列**
4. 在队列页开始下载

### 2）批量下载（播放列表/频道）

1. 输入 URL → 获取条目
2. 批量筛选与勾选
3. 设置统一下载策略与输出格式
   - 视频输出支持 `mp4 / mkv / webm`
   - 可启用“手动设置”，基于样本视频配置批量规则
4. 添加到队列 → 开始下载

## 🎯 下载策略（预设）

| 预设 | 说明 | yt-dlp 实际格式 |
|------|------|----------------|
| 最佳画质 | 最高画质与音质组合 | `bestvideo*+bestaudio/best` |
| 最佳兼容 | 兼容性最强（最高 1080p MP4） | `bestvideo[height<=1080][ext=mp4]+bestaudio[ext=m4a]/best[height<=1080]` |
| 最高 1080p | 限制最高 1080p | `bestvideo[height<=1080][ext=mp4]+bestaudio[ext=m4a]/best[height<=1080]` |
| 最高 4K | 限制最高 4K | `bestvideo[height<=2160]+bestaudio/best[height<=2160]` |
| 仅音频 | 只下载音频 | `bestaudio[ext=m4a]/bestaudio` |
| 最小体积 | 更小体积 | `best[height<=480]/worst` |
| 保留原始编码 | 保留原始编码组合 | `bestvideo+bestaudio/best` |
| HDR 优先 | 优先 HDR | `bestvideo[dynamic_range=HDR]+bestaudio/best` |
| 高帧率优先 | 优先 50fps+ | `bestvideo[fps>=50]+bestaudio/best` |
| 手动选择格式 | 手动选择格式 ID | 由格式表选择 |

### 批量手动策略

- 入口：批量页“启用手动” + “手动设置”
- 可先获取一个样本视频的格式，再把规则应用到整批任务
- 当前支持：
  - 预设1
  - 预设2（可选）
  - 最终兜底链（可选）
- 当前可配置的视频维度：
  - 高度
  - 编码偏好：`h264 / av1 / vp9`
  - 视频流容器偏好：`mp4 / webm`
  - 音频模式：`default / no_audio`
- 兜底顺序：
  - 同高度
  - 更高可用
  - 更低可用
  - 最后回退到 `bestvideo/best`
- 详细使用说明：
  - 适用范围：仅用于批量页的播放列表 / 频道下载，不影响单视频页的普通“手动选择格式 ID”流程。
  - 样本 URL：用于读取一个代表性视频的格式集合，供规则配置使用；它不是把样本视频的固定 `format_id` 直接复制到所有任务。
  - 取已选样本：会把当前已选条目的 URL 带入样本框，但仍需手动点击“获取格式”。
  - 获取格式：成功后才会刷新“高度 / 编码 / 视频流容器”的候选值；如果样本 URL 改了，旧结果立即作废，必须重新获取。
  - 预设1：第一层规则，启用手动后必须有效；至少要设置一个视频条件，不能三项都为“任意”。
  - 预设2：第二层规则，仅在预设1未命中时继续尝试；推荐把它设得比预设1更宽松。
  - 最终兜底链：第三层收尾规则；若启用了预设2，则以预设2为基础扩展，否则以预设1为基础扩展。
  - 高度：表示目标视频高度。预设层内优先尝试同高度；兜底链开启后，再按更高、再更低继续找可用格式。
  - 编码偏好：`h264 / av1 / vp9`。这里是“偏好顺序”，不是绝对锁死某一种编码；系统会先试你选择的编码，再试其他受支持编码。
  - 视频流容器偏好：`mp4 / webm`。这是源视频流的容器偏好，不等于批量页顶部的最终输出格式。
  - 音频模式：`default` 表示为该层视频规则追加 `+bestaudio`；`no_audio` 表示只下载视频流。
  - 重要区别：顶部“输出格式”控制最终封装方向；这里的“视频流容器”控制选流时优先找哪类视频流。
  - 推荐做法：`预设1` 放最想要的规格，`预设2` 放次优规格，列表来源较杂时建议开启“最终兜底链”提高成功率。
  - 当前限制：音频高级筛选暂不支持码率/扩展名细分；实际命中的具体格式 ID 以运行时日志为准。
